# 승인 시스템 테스트 방법

## 1. 데이터베이스 마이그레이션 적용

### Supabase Dashboard에서 실행

1. [Supabase Dashboard](https://supabase.com/dashboard) 접속
2. 프로젝트 선택 → SQL Editor 클릭
3. `supabase/migrations/021_add_approval_system.sql` 내용 복사하여 실행

---

## 2. 슈퍼관리자 계정 설정

### 첫 번째 사용자를 슈퍼관리자로 지정

```sql
-- 본인의 이메일로 계정 찾은 후 ID 확인
SELECT id, email FROM profiles WHERE email = '본인이메일@example.com';

-- 슈퍼관리자 권한 부여
UPDATE profiles SET is_super_admin = true WHERE id = '위에서_찾은_ID';
```

---

## 3. 관리자 앱 테스트

### 3.1 관리자 가입 신청 흐름

1. **앱 실행**
   ```bash
   cd apps/admin-app
   npm run build
   npx cap copy
   npx cap sync android
   npx cap open android
   ```

2. **카카오/구글 로그인**
   - [ ] 로그인 버튼 클릭 시 `admin-join.html`로 이동
   - [ ] 로그인 완료 후 자동으로 가입 페이지 진입

3. **가입 정보 입력**
   - [ ] 1단계: 기본 정보
     - 공부방/학원명 입력 (필수)
     - 대표자 성함 입력 (필수)
     - 연락처 입력 (필수, 자동 포맷)
   - [ ] 2단계: 사업자등록번호
     - 사업자등록번호 직접 입력 (000-00-00000)
     - **OCR 테스트**: 사업자등록증 이미지 업로드 후 "OCR 추출" 클릭
       - [ ] 번호가 자동으로 입력되는지 확인
     - [ ] 직접 수정 가능한지 확인
   - [ ] 3단계: 확인
     - 입력한 정보 표시 확인
   - [ ] 가입 신청 버튼 클릭 시 `admin-status.html`로 이동

4. **승인 대기 상태**
   - [ ] 대기중 상태 표시 (배경: 주황색)
   - [ ] 신청 정보 표시
   - [ ] 새로고침 버튼 동작

### 3.2 슈퍼관리자 승인 흐름

1. **슈퍼관리자 페이지 접속**
   - 웹 브라우저에서: `https://your-domain.com/apps/admin-app/www/super-admin.html`
   - 또는 앱에서 index.html 메뉴의 "슈퍼관리자" 클릭

2. **관리자 승인**
   - [ ] "관리자 신청" 탭에서 대기 중인 신청 목록 확인
   - [ ] 사업자등록증 이미지 확인 (클릭 시 확대)
   - [ ] 국세청 확인 버튼 동작
   - [ ] 승인 버튼 클릭 시 "승인 완료" 메시지
   - [ ] 거절 버튼 클릭 시 사유 입력 후 거절 완료

3. **승인 후 확인**
   - [ ] 관리자 앱에서 franchise.html 접근 가능
   - [ ] 메인 화면에서 슈퍼관리자 메뉴 표시

---

## 4. 학부모 앱 테스트

### 4.1 학부모 가입 신청 흐름

1. **앱 실행**
   ```bash
   cd apps/parent-app
   npm run build
   npx cap copy
   npx cap sync android
   npx cap open android
   ```

2. **카카오/구글 로그인**
   - [ ] 로그인 후 `parent-join.html`로 자동 이동

3. **학원 검색**
   - [ ] 2글자 이상 입력 시 자동 검색
   - [ ] 승인된 학원만 검색됨
   - [ ] 선택 시 선택 표시 (녹색 박스)

4. **자녀 정보 입력**
   - [ ] 학생 이름 (필수)
   - [ ] 생일 (필수)
   - [ ] 학교 (필수)
   - [ ] 학년 (필수)
   - [ ] 연락처 (필수)

5. **가입 신청**
   - [ ] 신청 완료 시 `parent-status.html`로 이동

### 4.2 승인 대기 상태

- [ ] 대기중 상태 표시
- [ ] 신청한 자녀 목록 표시
- [ ] 새로고침으로 상태 확인

### 4.3 여러 자녀 등록

- [ ] 첫 번째 자녀 승인 완료 후
- [ ] 다시 로그인 시 바로 메인 화면 진입
- [ ] 추가 자녀 등록 가능 (별도 기능으로 구현 필요 시)

---

## 5. 관리자 앱 - 학부모 승인 테스트

### 5.1 학부모 승인 탭

1. **franchise.html 접속**
   - 학생관리 탭 클릭
   - "승인대기" 서브탭 클릭

2. **승인 대기 목록**
   - [ ] 대기 중인 학생 목록 표시
   - [ ] 학생 정보 (이름, 학교, 학년, 생일, 연락처)
   - [ ] 대기 수량 배지 표시

3. **승인 처리**
   - [ ] 승인 버튼 클릭 시 승인 완료
   - [ ] 거절 버튼 클릭 시 사유 입력 모달 표시
   - [ ] 거절 사유 입력 후 거절 완료

4. **처리 완료 목록**
   - [ ] 승인/거절된 학생 목록 표시
   - [ ] 거절 사유 표시

## 6. 테스트 시나리오

### 시나리오 1: 관리자 가입 → 승인 → 서비스 이용

| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 관리자 앱 설치 및 로그인 | 가입 정보 입력 페이지로 이동 |
| 2 | 가입 정보 입력 후 신청 | 승인 대기 상태 페이지로 이동 |
| 3 | 슈퍼관리자가 승인 처리 | 승인 완료 메시지 |
| 4 | 관리자 앱 새로고침 | franchise(학생관리) 페이지 진입 가능 |

### 시나리오 2: 학부모 가입 → 승인 → 서비스 이용

| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 학부모 앱 설치 및 로그인 | 학원 검색 페이지로 이동 |
| 2 | 학원 선택 및 자녀 정보 입력 후 신청 | 승인 대기 상태 페이지로 이동 |
| 3 | 관리자가 승인 처리 | 승인 완료 메시지 |
| 4 | 학부모 앱 새로고침 | 메인 화면 진입 가능, 출결 확인 가능 |

### 시나리오 3: 거절 → 재신청

| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 관리자가 가입 신청 거절 (사유 포함) | 거절 상태, 사유 표시 |
| 2 | "다시 신청하기" 클릭 | 가입 정보 입력 페이지로 이동 |
| 3 | 정보 수정 후 재신청 | 다시 승인 대기 상태 |

## 8. 문제 해결

### OCR이 작동하지 않을 때

1. Tesseract.js 로드 확인 (브라우저 콘솔)
2. 이미지 파일이 너무 크거나 흐릿한지 확인
3. 직접 입력 필드에 번호 입력 가능

### Supabase RLS 정책 오류

```sql
-- RLS 정책 확인
SELECT * FROM pg_policies WHERE tablename IN ('profiles', 'students');

-- RLS 다시 적용
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE students ENABLE ROW LEVEL SECURITY;
```

### 푸시 알림이 안 올 때

1. FCM 토큰이 등록되었는지 확인
2. Supabase Edge Function이 배포되었는지 확인
3. 앱 권한 설정 확인
